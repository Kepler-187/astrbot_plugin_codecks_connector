# 命令参考

所有命令使用 `/codecks` 或 `/ck` 前缀。

---

## 📂 项目

### `/ck projects`
**别名**：`/ck 项目`

列出组织中所有项目，显示名称、序号和 ID。

**示例输出**：
```
📂 项目列表 (3 个)
━━━━━━━━━━━━━━━━
  #1 🔒 主项目  (ID: abc123)
  #2 🔒 美术资源  (ID: def456)
  #3 🔒 QA 测试  (ID: ghi789)
```

---

## 📦 卡组

### `/ck decks [项目ID]`
**别名**：`/ck 牌组`

列出卡组。不指定项目 ID 时列出所有卡组。

**示例**：
```
/ck decks              ← 所有卡组
/ck decks abc123       ← 指定项目的卡组
```

---

## 🃏 卡片

### `/ck cards [卡组ID] [数量]`
**别名**：`/ck 卡片`

列出卡片，默认按创建时间降序，默认显示 10 张。

**示例**：
```
/ck cards              ← 最新 10 张卡片
/ck cards abc123       ← 指定卡组的卡片
/ck cards abc123 20    ← 指定卡组的前 20 张
```

### `/ck card <ID>`

查看单张卡片的详细信息，包括标题、状态、优先级、工作量、截止日期等。

**示例**：
```
/ck card xyz789
```

### `/ck search <关键词>`
**别名**：`/ck 搜索`

全文搜索卡片内容。

**示例**：
```
/ck search 登录bug
/ck 搜索 UI优化
```

### `/ck newcard <标题> [卡组ID] [工作量] [优先级]`
**别名**：`/ck 新建卡片`、`/ck 创建卡片`

创建新卡片。

**参数说明**：
| 参数 | 是否必填 | 说明 |
|---|---|---|
| 标题 | ✅ | 卡片标题 |
| 卡组ID | ❌ | 放入指定卡组（不填则创建到手牌） |
| 工作量 | ❌ | 工作量数值，默认 0 |
| 优先级 | ❌ | a/b/c/d，默认 c |

**示例**：
```
/ck newcard "修复登录闪退"
/ck newcard "优化加载速度" abc123 5 b
/ck 新建卡片 "添加新手引导" def456
```

### `/ck complete <ID>`
**别名**：`/ck 完成`

将卡片标记为已完成。

### `/ck reopen <ID>`
**别名**：`/ck 重开`、`/ck 重新打开`

重新打开已完成的卡片。

### `/ck update <ID> <字段> <值>`
**别名**：`/ck 更新`

更新卡片的指定字段。

**支持的字段**：

| 字段 | 含义 | 示例值 |
|---|---|---|
| `title` / `content` | 标题/内容 | `"新标题"` |
| `effort` | 工作量 | `5` |
| `priority` | 优先级 | `a` / `b` / `c` / `d` |
| `duedate` | 截止日期 | `2025-12-31` |

**示例**：
```
/ck update xyz789 priority a
/ck update xyz789 effort 8
/ck 更新 xyz789 duedate 2025-06-30
```

### `/ck assign <卡片ID> <用户ID>`
**别名**：`/ck 分配`

将卡片分配给指定用户。用户 ID 可通过 `/ck users` 查看。

### `/ck unassign <ID>`
**别名**：`/ck 取消分配`

取消卡片的负责人分配。

### `/ck comment <ID> <内容>`
**别名**：`/ck 评论`

为卡片添加评论。

**示例**：
```
/ck comment xyz789 "已确认复现，正在处理"
```

### `/ck setmilestone <卡片ID> <里程碑ID>`
**别名**：`/ck 设置里程碑`

设置卡片所属的里程碑。里程碑 ID 可通过 `/ck milestones` 查看。

### `/ck clearmilestone <ID>`
**别名**：`/ck 清除里程碑`

清除卡片的里程碑关联。

---

## 🏁 里程碑

### `/ck milestones`
**别名**：`/ck 里程碑`

列出所有里程碑（已过滤已删除的）。

---

## 👥 用户

### `/ck users`
**别名**：`/ck 用户`、`/ck 成员`

列出组织成员，显示名称和 ID（ID 用于 assign 等操作）。

### `/ck me`

查看当前 Token 对应的用户信息。

### `/ck hand`
**别名**：`/ck 手牌`、`/ck 我的任务`

查看当前用户的手牌（未完成的个人任务）。

---

## 🏷️ 标签 & 冲刺

### `/ck tags [项目ID]`
**别名**：`/ck 标签`

列出标签。可选按项目筛选。

### `/ck sprints [项目ID]`
**别名**：`/ck 冲刺`

列出冲刺。可选按项目筛选。

---

## 📊 统计

### `/ck stats [项目ID]`
**别名**：`/ck 统计`

查看卡片统计信息：总数、完成数/率、进行中、待办、各状态工作量。

---

## ⚙️ 系统

### `/ck help`
**别名**：`/ck 帮助`

显示命令快速参考。

### `/ck config`

查看当前插件配置状态（Token 是否已设置、子域名等）。

### `/ck debug`

测试与 Codecks API 的连接，成功后显示组织名称。
