# 常见问题 (FAQ)

## ❌ 连接失败: 认证失败

**原因**：Token 无效或已过期。

**解决**：
1. 重新从浏览器获取 Token（登录 Codecks → F12 → Network → 找 `at` cookie）
2. 注意：如果你登出了 Codecks，Token 会立即失效
3. 确认 Token 中没有多余的空格或换行

## ❌ 连接失败: 服务器内部错误 (500)

**原因**：多数情况下是查询了不存在的字段名。Codecks API 对无效字段不会报具体错误，而是直接返回 500。

**解决**：
1. 确保使用最新版插件（已修复已知的字段名问题）
2. 如果使用 raw query 功能，检查字段名是否拼写正确
3. 可以用浏览器开发者工具观察 Codecks 网页版的实际 API 请求来确认有效字段名

## ❌ 请求过于频繁

**原因**：超过速率限制（40 请求/5秒/IP）。

**解决**：
1. 插件已内置自动重试，正常情况下不会触发
2. 可以在配置中增大 `rate_limit_delay`（如设为 0.3）
3. 避免短时间内连续发送大量命令

## 📭 查询结果为空

**可能原因**：
1. 组织中确实没有对应数据
2. Token 对应的用户没有相关项目/卡组的访问权限
3. 数据已被删除或归档

**排查**：
1. 用 `/ck debug` 确认连接正常
2. 用 `/ck projects` 确认能看到项目列表
3. 在 Codecks 网页版确认数据存在

## 卡片创建后找不到

**原因**：未指定 `卡组ID` 时，卡片会创建到手牌（hand）。

**解决**：
1. 用 `/ck hand` 查看手牌
2. 创建时指定卡组ID：`/ck newcard "标题" <卡组ID>`

## Token 安全问题

- Token 等同于你的用户身份
- 建议创建一个专用的 Observer 用户用于 API 集成
- 不要将 Token 提交到公开仓库
- 如果怀疑 Token 泄露，立即在 Codecks 中重新登录（旧 Token 自动失效）

## 如何获取卡片/用户/项目的 ID？

所有查询命令都会在结果中显示项目的 ID。例如：

```
/ck projects      → 显示每个项目的 ID
/ck users         → 显示每个用户的 ID
/ck cards         → 显示每张卡片的 ID
/ck milestones    → 显示每个里程碑的 ID
```

将这些 ID 用于操作命令（如 assign、setmilestone 等）。

## 支持哪些聊天平台？

插件通过 AstrBot 框架运行，理论上支持 AstrBot 支持的所有平台（QQ、微信、Telegram 等），不依赖特定平台 API。
